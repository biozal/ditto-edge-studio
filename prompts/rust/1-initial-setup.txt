  You can see it now requires 1.91.1.  For now I want to focus on the starting screen, which in SwiftUI is the Database Listing and Database Editor views which are launched from the ContentView.

  The Database Listing view allows you to see a listing of Database configurations that are registered into the SQLite local database from either the Database Editor form, or you can scan in a configuration using the Laptop's camera to scan in a QR Code.  This screen has to have the ability to also generate a QR Code of the configuration in order to display it for other devices to import in.  

  A screenshot of the Database Listing screen can be found here:
  database-list.png

  Functionally I think this UI works.  The left side is attached to the bottom of the screen and gives some buttons along with the name of the app.  The + Database Config button brings up the Database Editor form.  The Ditto Portal button opens a web browser and navigates to the site:  https://portal.ditto.live/

  The Import from QR Code brings up a camera if the hardware is available - like the laptop camera on a Macbook or Windows laptop, and an error message if it can't detect a camera.  It then waits for a QR Code to be presented and scans that QR Code, validates it's a proper Ditto Database Configuration and then adds that configuration to the SQLite database and updates the Database Listing.  

  The right side of Database Listing UI is a list of Databases that are registered in the database.  These show the name of the registeration along with the databaseId and a database icon.  

  A user can right click on a database configuration and a menu pops up - you can see this in the database-list-right-click.png screenshot.  This allows the following actions:

- Edit allows you to edit this database configuration in the database editor form.

- Show QR Code takes the database configuration and displays it in a QR Code.  It needs to be compatible with the Android and SwiftUI versions of the app.  

- Delete allows you to delete the database configuration from the SQLite database AND remove all the files associated with it on disk.
