# DQL Strict Mode

## What Is DQL Strict Mode?

`DQL_STRICT_MODE` is a Ditto system setting that controls how nested objects in documents are typed by default.

| Setting | Nested objects typed as | Merge behavior |
|---------|------------------------|----------------|
| `false` (default for SDK 5.0) | `MAP` | Field-level merging — individual keys can be updated independently |
| `true` (default for SDK 4.x) | `REGISTER` | Last-write-wins on the entire object — overwrites the whole nested object |

## Default Value

**SDK 5.0 and later:** `DQL_STRICT_MODE = false` (MAP semantics)
**SDK 4.x:** Effectively `DQL_STRICT_MODE = true` (REGISTER semantics)

Edge Studio stores and applies this setting per-database and **defaults to `false`**, which is correct for most SDK 5.0 users.

## When Should I Enable Strict Mode?

Enable `DQL_STRICT_MODE = true` only if:

1. **SDK 4.x compatibility** — You are syncing with peers running Ditto SDK 4.x and need identical merge behavior.
2. **Explicit REGISTER semantics** — Your data model intentionally uses last-write-wins for nested objects.

In all other cases, leave strict mode **disabled**.

## Cross-Peer Warning

> ⚠️ **Mismatched strict mode settings across peers may cause data loss.**

If one peer has strict mode **enabled** and another has it **disabled**, nested object fields may appear missing after sync. This happens because:
- Strict mode (REGISTER) treats the entire nested object as one atomic value
- Non-strict mode (MAP) treats each nested key independently

**All peers syncing the same database should use the same strict mode setting.**

## How to Configure in Edge Studio

### macOS / iPadOS

1. Open Edge Studio
2. From the database list, tap **Edit** on any database
3. Scroll to **Developer Options**
4. Toggle **Enable DQL Strict Mode**
5. Tap **Save**

The setting takes effect immediately the next time the database is opened (before `startSync()` is called).

### Android

1. Open Edge Studio for Android
2. From the database list, tap the edit icon on any database
3. Scroll to **Developer Options**
4. Toggle **Enable DQL Strict Mode**
5. Tap **Save**

## QR Code Sharing

When you export a database configuration as a QR code, `isStrictModeEnabled` is included in the payload. Scanning the QR code on another device will automatically apply the same strict mode setting.

**Backward compatibility:** QR codes generated by older versions of Edge Studio (before this feature was added) will default to `isStrictModeEnabled = false` when scanned by newer versions.

## Technical Details

The setting is applied using the Ditto DQL system command:

```sql
ALTER SYSTEM SET DQL_STRICT_MODE = false
```

This command must be executed **before** `startSync()` is called. Edge Studio handles this automatically when opening a database.
